1-3. Variable and Data-type :

Python code: <%# Copy source code %> 
variable_name = 'value' # no command for declaring a variable
# python variable names are "case-sensitive"
py_x = 1
py_X = 1 # they are two different variable
# Python is dynamically but strongly typed
py_string = 'string'
py_number = 123
py_boolean = True # Capitalize
# There is no undefine in Python
py_none_value = None # equal to null in Javascript
"""
There are four collection data types in Python :
List       : ordered and changeable.            Allows duplicate members.
Tuple      : ordered and unchangeable.          Allows duplicate members.
Set        : unordered and unindexed.           No duplicate members.
Dictionary : unordered, changeable and indexed. No duplicate members.
the index of values start from [0]
There are some difference between each other,
we will mention it later in Module 3. Object
"""
py_list = ['a', 'b', 'c', 1, 2, 3] # like the array in Javascript
py_tuple = ('a', 'b', 'c', 1, 2, 3)
py_set = {'a', 'b', 'c', 1, 2, 3}
py_dictionary = {
    'a' : 'apple',
    'b' : 'banana',
    'c' : 'cherry',
    'year' : 2019
}
Javascript code: <%# Copy source code %> 
var variableName = 'value';
// Javascript variable names are "case-sensitive"
var jsx = 1;
var jsX = 1; // they are two different variable
// Javascript is dynamically and loosely typed
var jsString = 'string'; // type of string
var jsNumber = 123; // type of number
var jsBoolean = true; // type of boolean
var jsUndefine; // value is undefine
var jsNullValue = null; // value is null
var jsArray = ['value', 1, 2, 3, '四', '五']; // type of array
// the index of values start from [0]
// arrays are used to store multiple values in single variable
var jsObject = {
    key : 'value',
    name : 'Mary',
    age : 20 
} // type of object

"Dynamically typed language" on the other hand means that you do not need to declare variable type (and in most cases variable itself) before use. In such a languahe variable will get type after it is assigned with a value gor the first time.

"Strong typing" means that the type of a value doesn't change in unexpected ways. A string containing only digits doesn't magically become a number (e.g. string and integer.). Every change of type requires an explicit conversion.

"Loosely typed" means language does not bother with types too much, and does conversions automatically, i.e. you can easily add string to an integer and get the result as a string.
